## 1. 시스템 접속

### 1.1 웹 브라우저 접속

```
https://<사내 호스트>/presentation-evaluator/
```

**지원 브라우저**:
- Chrome 100+ (권장)
- Edge 100+
- Safari 15+
- Firefox 100+

### 1.2 Android 태블릿

- 사내 MDM(모바일 디바이스 관리)을 통해 배포된 APK 설치
- 홈 화면의 "제품 설명회 평가" 아이콘 실행

### 1.3 로그인

| 항목 | 입력 |
|------|------|
| 관리자 ID | 사내 발급 ID |
| 비밀번호 | 초기 비밀번호 (첫 로그인 시 변경 권장) |

> ⚠️ **3회 연속 실패 시 계정 잠금** — 정보보호팀에 문의

## 2. 관리자 가이드

### 2.1 평가 템플릿 만들기

평가는 **평가 템플릿**을 먼저 만든 뒤, 설명회에 적용하는 구조입니다.

#### 단계
1. 좌측 메뉴 → **평가 템플릿** 클릭
2. 우측 상단 **+ 템플릿 추가** 버튼
3. 템플릿 정보 입력:
   - **템플릿명**: 예) "2026년 1분기 핀테크 평가"
   - **기본 템플릿 여부**: 체크 시 신규 설명회에 자동 선택됨
4. **평가 항목 추가**:
   - 항목명 (예: "기술 우수성")
   - 배점 (예: 20점)
   - 설명 (선택)
5. 최소 1개 이상 항목 추가 → **저장**

#### 권장 평가 항목 예시 (금융권 IT 제품 평가용)

| 항목 | 배점 | 설명 |
|------|------|------|
| 기능 적합성 | 25 | 요구사항 충족도 |
| 보안성 | 25 | 금융권 보안 기준 부합 |
| 안정성 | 20 | 장애 대응, SLA |
| 가격 적정성 | 15 | TCO 관점 |
| 도입·연동 용이성 | 10 | 기존 시스템 통합 |
| 벤더 신뢰성 | 5 | 회사 평판, 레퍼런스 |
| **합계** | **100** | |

---

### 2.2 평가자 등록

#### 단계
1. 좌측 메뉴 → **평가자 관리**
2. **+ 평가자 추가** 버튼
3. 정보 입력:
   - **이름** (필수)
   - **부서** (필수)
   - **소속** (선택, 예: "여신심사본부")
4. **저장**

> 💡 평가자는 **여러 설명회에 재사용** 가능합니다. 한 번만 등록하세요.

#### 평가자 비활성화

- 평가자 카드 우측 메뉴 → **비활성화**
- 비활성화된 평가자는 신규 설명회 등록 시 선택 불가
- 기존 평가 이력은 보존됨 (감사 추적)

---

### 2.3 설명회 생성

#### 단계
1. 좌측 메뉴 → **설명회 관리**
2. 우측 상단 **설명회 생성** 버튼
3. 입력 폼:
   - **설명회 제목** (필수): "2026년 4분기 핀테크 제품 설명회"
   - **제품명** (필수): "디지털 자산 관리 플랫폼 v2.0"
   - **발표자** (필수): "홍길동"
   - **발표사** (선택): "(주)핀테크솔루션"
   - **설명회 일시** (필수): 캘린더 선택
   - **평가 템플릿** (필수): 라디오 버튼 선택 (기본 템플릿이 자동 선택됨)
   - **평가자 등록** (필수, 최소 1명): 체크박스 선택
   - **메모** (선택): 관리자 내부 메모
4. **생성** 버튼

> ⚠️ 평가 템플릿이나 평가자가 없으면 **먼저 등록**해야 합니다.

#### 생성 후 상태

생성된 설명회는 **진행 예정** 상태로 등록되며, 다음과 같은 정보를 확인할 수 있습니다:
- 평가자별 인증 코드 (또는 QR)
- 평가 진행률 (0/N)

---

### 2.4 설명회 진행

#### 진행 시작
1. 설명회 카드 클릭 → 상세 화면 진입
2. **설명회 시작** 버튼 → 상태가 **진행 중**으로 변경
3. 평가자들에게 인증 코드 또는 QR 코드 공유:
   - **인증 코드 방식**: 평가자별 6자리 코드를 종이/메신저로 전달
   - **QR 방식**: 화면의 QR을 평가자가 스캔

#### 진행 중 모니터링
- 평가자별 제출 현황 실시간 확인 (X/N 명 완료)
- 미제출자 인증 코드 재전송 가능

#### 진행 종료
- **설명회 종료** 버튼 → 상태가 **종료**로 변경
- 더 이상 신규 평가 입력 불가

---

### 2.5 결과 집계 및 잠금

#### 결과 화면 진입
1. 종료된 설명회 카드 클릭
2. 우측 상단 **결과 보기** 버튼

#### 표시 정보
- **종합 점수**: 평균 / 표준편차 / 최고·최저
- **항목별 점수**: 각 평가 항목의 평균과 분포
- **평가자별 상세**: 개별 점수와 코멘트
- **차트**: 막대그래프, 레이더 차트

#### 결과 잠금 (중요!)

> 🔒 **잠금 후에는 평가 결과를 절대 수정할 수 없습니다.**
> 결과를 사내 의사결정 자료로 사용하기 전 반드시 **잠금** 처리하세요.

##### 잠금 절차
1. 결과 화면 우측 상단 **잠금** 버튼
2. 확인 다이얼로그에서 **확인** 클릭
3. 잠금 사유 입력 (감사 로그에 기록됨)

##### 잠금 후 표시
- 설명회 카드에 🔒 아이콘 표시
- "잠금: YYYY-MM-DD HH:MM by 관리자명" 표시

---

### 2.6 CSV 내보내기

#### 단계
1. 결과 화면 우측 상단 **CSV 내보내기** 버튼
2. 다운로드 파일 형식:
   ```
   {설명회제목}_{날짜}.csv
   예) 2026Q4_핀테크설명회_20260530.csv
   ```

#### CSV 컬럼 구조

| 컬럼 | 내용 |
|------|------|
| `evaluator_id` | 평가자 ID |
| `evaluator_name` | 평가자 이름 |
| `department` | 부서 |
| `question_1_score` | 평가 항목 1 점수 |
| ... | ... |
| `total_score` | 합계 점수 |
| `comment` | 종합 의견 |
| `submitted_at` | 제출 시각 |

> 💡 CSV 파일은 Excel에서 직접 열 수 있으며, UTF-8 인코딩입니다.

## 3. 평가자 가이드

> **v1.0.1 변경사항 (2026-06-04)**
> 평가자 인증이 **입장코드 단독 진입** 방식으로 단순화되었습니다.
> 기존의 4단계 인증(설명회ID + 이메일 + 생년월일 + 입장코드)은 폐기되었으며,
> 이제 관리자가 발급한 **입장코드 1개만 입력**하면 즉시 평가 화면으로 진입합니다.
> (1회용 코드 + 만료 시간으로 보안 확보)

### 3.1 평가 진입

#### 방법 A: 입장코드 입력 (권장)
1. 평가용 디바이스에서 시스템 접속
2. 메인 화면 **평가 참여** 버튼
3. 관리자가 전달한 **입장코드 8자리** 입력
   - 형식: `XXXX-XXXX` (예: `A3K9-M2P7`)
   - 영문 대문자 + 숫자 (혼동 방지를 위해 0, 1, O, I, L은 사용하지 않음)
   - 입력 시 자동 대문자 변환
4. **[입장하기]** 버튼 클릭 → 평가 화면으로 즉시 진입

#### 방법 B: QR 스캔
1. 관리자가 전달한 QR 코드를 카메라로 스캔
2. 입장코드가 자동 입력되어 즉시 평가 화면 진입
3. QR에는 평가자별 개별 입장코드가 포함되어 있어 별도 입력 불필요

#### 인증 실패 시 안내 메시지

| 메시지 | 원인 | 해결 |
|--------|------|------|
| 유효하지 않은 입장코드입니다 | 코드 오타 또는 발급된 적 없음 | 관리자에게 코드 재확인 요청 |
| 만료된 입장코드입니다 | 설명회 종료 시간 경과 | 관리자에게 재발급 요청 |
| 이미 사용된 입장코드입니다 | 1회용 코드 재사용 시도 | 관리자에게 재발급 요청 |
| 아직 시작되지 않은 설명회입니다 | 관리자가 진행 시작 전 | 관리자에게 진행 시작 요청 |
| 이미 종료된 설명회입니다 | 설명회 종료 후 접속 | 평가 마감 → 재참여 불가 |

### 3.2 평가 입력

1. 평가 항목별로 점수 입력
   - 슬라이더 또는 숫자 입력
   - 배점 범위 초과 시 자동 제한
2. **종합 의견** 입력 (선택)
3. **임시 저장** 또는 **최종 제출**

> 💡 **임시 저장**: 입력 중 중단 시 활용 (재진입 가능)
> ⚠️ **최종 제출**: 제출 후 수정 불가

### 3.3 평가 완료

- 제출 완료 메시지 표시
- 본인 평가 내용 확인 가능 (집계 잠금 전까지)
- 다른 평가자의 점수는 **비공개**

### 3.4 입장코드 보안 원칙

| 원칙 | 설명 |
|------|------|
| **1회용 (Single-use)** | 입장코드는 한 번 사용 후 즉시 무효화 |
| **만료 시간** | 설명회 종료 시간까지만 유효 |
| **평가자별 고유** | 각 평가자마다 다른 입장코드 발급 (공유 금지) |
| **본인확인 전제** | 코드 발급 시점에 관리자가 평가자 본인확인 완료한 것으로 간주 |
| **감사 로그** | 사용/실패 시도 모두 감사 로그에 기록 |

## 4. 감사 로그 활용

### 4.1 감사 로그 조회

1. 좌측 메뉴 → **감사 로그**
2. 시간순 정렬된 액션 목록 표시

### 4.2 필터링

- **기간 필터**: 시작일 ~ 종료일
- **액션 유형**: 로그인, 생성, 수정, 삭제, 잠금, 내보내기 등
- **수행자 필터**: 특정 관리자만 조회

### 4.3 감사 시나리오 예시

#### "특정 설명회의 모든 활동 추적"
1. 감사 로그 진입
2. 검색창에 **설명회 ID 또는 제목** 입력
3. 시간순 활동 내역 확인:
   - 생성 → 수정 → 평가자 추가 → 진행 → 종료 → 잠금 → 내보내기

#### "비정상 로그인 시도 확인"
1. 액션 필터 → **LOGIN_FAILED**
2. 동일 ID에서 짧은 시간 내 다수 실패 패턴 확인

## 5. FAQ

### Q1. 설명회 생성 후 평가자를 추가/변경할 수 있나요?
**A.** 진행 시작 전(상태=진행 예정)까지는 자유롭게 수정 가능합니다.
진행 중에도 평가자 추가는 가능하지만, 제거는 권장하지 않습니다 (감사 추적 어려움).

### Q2. 결과 잠금을 실수로 했습니다. 해제할 수 있나요?
**A.** ❌ **불가능합니다.** 잠금은 무결성 보장을 위한 기능이며,
해제하려면 시스템 관리자가 데이터베이스 직접 수정해야 하며 감사 대상입니다.

### Q3. 평가자가 입장코드를 분실했습니다.
**A.** 관리자 화면에서 **설명회 상세 → 초대 탭**에서 평가자별 입장코드를 다시 확인할 수 있습니다.
- 카드를 탭하면 QR + 입장코드 + 복사 버튼이 나타납니다.
- **[초대문구 복사]** 버튼으로 평가자명/설명회/입장코드/만료시간을 함께 복사 후 전달 가능합니다.
- 이미 사용한 코드인 경우 관리자가 평가자 목록에서 재발급해야 하며, 재발급 이벤트는 감사 로그에 기록됩니다.

### Q4. 평가 데이터를 외부에 백업하고 싶습니다.
**A.** 다음 방법을 사용하세요:
- **개별 설명회**: CSV 내보내기
- **전체 데이터**: Hive 박스 파일 직접 복사 (사내 정책 확인 필요)

### Q5. 한 명의 평가자가 여러 설명회에 참여하면 어떻게 되나요?
**A.** 평가자는 한 번 등록하면 여러 설명회에 재사용됩니다.
각 설명회마다 **별도의 인증 코드**가 발급되므로 혼동 없습니다.

### Q6. 평가 점수가 너무 들쭉날쭉합니다. 평균 외 다른 지표를 볼 수 있나요?
**A.** 결과 화면에서 다음 지표를 확인할 수 있습니다:
- 평균 (Mean)
- 표준편차 (Standard Deviation)
- 최고·최저 제외 평균 (옵션)
- 항목별 분포 (히스토그램)

향후 v1.2에서 **이상치 자동 감지 (AI 기반)** 기능이 추가될 예정입니다.

### Q7. 시스템이 갑자기 느려졌습니다.
**A.** 다음을 확인하세요:
1. 디바이스 저장 공간 (Hive 박스 크기 확인)
2. 브라우저 캐시 클리어 (Ctrl+Shift+R)
3. 오래된 종료 설명회 아카이브 처리

### Q8. 동일한 평가 템플릿을 복사해서 일부만 수정하고 싶습니다.
**A.** 평가 템플릿 목록에서 우측 메뉴 → **복제** 기능을 사용하세요.
복제 후 이름을 변경하고 항목을 수정하면 됩니다.
